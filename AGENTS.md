# プロジェクト概要
このリポジトリは、Codex / Claude Code / Cursor 向けの skills, subagents, hooks, ドキュメント運用を開発・管理するためのリポジトリである。

# このリポジトリでの作業方針
- 回答・説明・コミットメッセージは日本語で行う。
- コミットメッセージは Conventional Commits 形式を使う。
- 構造的変更と振る舞いの変更を同一コミットに混ぜない。
- 変更はできるだけ小さな論理単位に分割する。
- 既存ファイルの命名規則と配置規則を優先する。
- 新しい依存関係の追加は、必要性を説明したうえで提案する。
- 破壊的な変更、削除、ネットワーク利用を伴う操作は、実行前に意図を明示する。

# ディレクトリの約束
- Codex 用の設定は `.codex/` 配下を正本とする。
- subagent は `.codex/agents/` に置く。
- skill は `.codex/skills/` に置く。
- hook は `.codex/hooks.json` と `.codex/hooks/` に置く。
- このリポジトリの拡張は、できるだけ再利用可能な単位で設計する。

# Rules
- skills, rules, subagents, hooks の新規作成・編集はプロジェクトディレクトリ内（.claude/）で行う。~/.claude/ への直接配置は禁止。
# Commit
- Conventional Commits 形式に従い、メッセージは日本語で書く（例: feat: ユーザー認証スキルを追加）。
- Tidy First に則り、構造的変更と振る舞いの変更を同一コミットに混ぜない。可能な限り小さな論理単位に分割してコミットする。

## 出力ルール
- ユーザー向けの最終回答には、`Running ...`、`Ran ...`、`Thinking ...` などの実行ログをそのまま貼り付けない。
- 調査過程を伝える必要がある場合は、実行ログの列挙ではなく、何を確認したかを1〜2文で要約する。
- ファイルやディレクトリを示すときは、ローカル絶対パスではなく、必ずリポジトリ相対パスを使う。
- 例: `C:\Users\name\repo\src\app.ts` ではなく `src/app.ts` と書く。
- 外部共有向けの説明では、作業ログよりも結論・根拠・該当ファイルを優先する。
