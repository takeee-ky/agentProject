---
name: skill-creator
description: 新しい Codex skill を設計・作成するときに使う。責務分解、名前付け、description 設計、SKILL.md の初稿作成まで行う。
---

# 目的

新しい skill を作るときの設計を安定化する。

# 入力

- 作りたい skill の目的
- 想定ユーザー
- 使う場面
- やらないこと
- 必要なら参考資料

# 手順

1. skill の責務を 1 つに絞る。
2. 明示的な入出力を定義する。
3. 使う場面と使わない場面を切り分ける。
4. description を短く具体的に書く。
5. 最小の SKILL.md を作る。
6. 必要なら references/ と scripts/ の追加を提案する。

# 出力

- skill 名
- description
- SKILL.md 初稿
- 必要な補助ファイル一覧

# 禁止事項

- 1 skill に複数責務を詰め込みすぎない。
- description を抽象的にしすぎない。
- 実行手順と設計思想を混同しない。
